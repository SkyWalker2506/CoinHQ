## Current State

Auth is complete: Google OAuth → JWT, `get_current_user` dependency on every protected endpoint, multi-user isolation via `profile.user_id == current_user.id` on all queries. Exchange adapter pattern in `backend/app/exchanges/` is consistent — each exchange implements `base.py`.

`_oauth_states` is a module-level dict (`auth.py:29`) used to validate OAuth `state` parameter against CSRF. On multi-replica Railway deployments or process restarts, the state generated by one instance cannot be verified by another — causing 100% login failure on scale-out or deploy.

Test suite exists but coverage is shallow: `test_portfolio_service.py`, `test_isolation.py`, `test_exchanges.py` cover happy paths. No failure mode tests, no API-level integration tests.

## Findings

### 🔴 Critical

**F1 — `check_exchange_limit` is dead code**
`backend/app/core/limits.py:25` — Function defined and correct. Zero call sites in the codebase. Free-tier users can add unlimited exchange keys via direct API calls. This is both an architecture (dead code) and a business logic (tier enforcement) bug.
Fix: add `await check_exchange_limit(current_user, db)` at `backend/app/api/v1/keys.py` before key creation.

**F2 — `_oauth_states` in-memory dict — single-instance architecture assumption**
`backend/app/api/v1/auth.py:29`
```python
_oauth_states: dict[str, float] = {}
```
State generated during `/auth/google` stored in process memory. Any blue/green deploy, crash, or horizontal scale-out invalidates all in-flight OAuth flows, returning "Invalid OAuth state — possible CSRF attack" to users mid-login. Not thread-safe across multiple uvicorn workers in the same process.
Fix: `redis.setex(f"oauth:state:{state}", 600, "1")` + `redis.get()` + `redis.delete()` pattern using `app.state.redis`.

### 🟡 Important

**F3 — Admin gate uses string comparison instead of `UserTier` enum**
`backend/app/api/v1/admin.py:15`
```python
if user.tier != "admin":
```
`UserTier.ADMIN` enum value exists at `models/user.py:10`. String comparison is fragile — a typo in either place silently breaks the gate. Fix: `if user.tier != UserTier.ADMIN:`.

**F4 — Auth callback creates two separate `httpx.AsyncClient` instances**
`backend/app/api/v1/auth.py:96-119` — Two separate `async with httpx.AsyncClient() as client:` blocks for (1) token exchange and (2) userinfo fetch, ignoring `app.state.http_client`. Creates 2 connection overheads per login. Fix: use `app.state.http_client` for both calls.

**F5 — Test coverage shallow, no API-level integration tests**
`tests/` has unit tests for services and isolation checks but no FastAPI `TestClient` integration tests that hit full route → service → DB → response paths. A regression in middleware, dependencies, or schema serialization would not be caught.

**F6 — `DEBUG` mode triggers `init_db()` bypassing Alembic**
`backend/app/main.py:30-31` — If `DEBUG=True` leaks to production (e.g., in a misconfigured deploy), SQLAlchemy `create_all()` runs instead of Alembic migrations, silently diverging the schema from migration history.

**F7 — No CI workflow**
`.github/workflows/` directory does not exist. PRs merge without automated `ruff`, `pytest`, or security checks. Fix: minimal CI with `uv run ruff check .`, `uv run pytest`, and `pnpm lint`.

### 🟢 Good

**F8 — Exchange adapter pattern well-implemented.** Each exchange inherits from `base.py`; factory pattern returns the correct adapter by name. Adding a new exchange requires only implementing `base.py`.

**F9 — `get_current_user` dependency consistently applied.** Grepping for `Depends(get_current_user)` returns hits on all 16 protected endpoints across 5 files — no auth bypass found.

**F10 — `structlog` JSON logging.** Backend logs are structured, machine-parseable, and the `_mask_key()` helper prevents plaintext key logging.

**F11 — `asyncio.gather` for parallel exchange fetch.** Portfolio aggregation no longer serially awaits each exchange.
